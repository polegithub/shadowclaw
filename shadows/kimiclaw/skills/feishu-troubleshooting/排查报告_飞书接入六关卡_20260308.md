# OpenClaw 接入 Kimi 与飞书 (Feishu) 综合排查与最佳实践报告

> 原始报告来自 Kimi龙虾（可聊天），2026-03-08 排查记录

---

## 第一部分：排查过程全复盘

在今天的调试中，我们一共遭遇并跨越了 6 个核心关卡。整个链路的故障自下而上覆盖了：配置死锁、版本冲突、依赖缺失以及大模型风控拦截。

### 1. 配置文件死锁 (Config Deadlock)

- **现象**：命令行执行任何操作直接崩溃，提示 `plugin not found: feishu` 和 `Invalid config`。
- **根因**：OpenClaw 的 CLI 工具在执行命令前会强制校验 `openclaw.json`。由于配置文件中提前写入了飞书配置，但系统尚未安装该插件，导致校验失败，形成"无法安装插件"的死锁。
- **解决**：临时从配置文件中删除了 `feishu` 节点，恢复 CLI 运行能力后，再进行插件安装。

### 2. SDK 路径变更导致模块丢失

- **现象**：报错 `Cannot find module '.../plugin-sdk/index.js/feishu'`。
- **根因**：OpenClaw 近期重构了底层 SDK 目录结构，将入口统一打包为 `index.js`，而旧版插件代码仍在使用旧的相对路径进行 import。

### 3. 核心 API 破坏性变更 (Breaking Change)

- **现象**：报错 `createFixedWindowRateLimiter is not a function`。
- **根因**：这是本次排查的最核心转折点。系统当前的 OpenClaw 版本为 `2026.2.13`，而安装的飞书插件（`@m1heng-clawd/feishu`）要求核心版本必须 `>=2026.3.1`。旧核心缺少新版插件所需的底层函数。
- **解决**：果断放弃手动修改源码，直接将 OpenClaw 全局升级至最新版（`>=2026.3.1`），彻底消除了所有代码层面的不兼容报错。

### 4. 幽灵依赖与重复插件警告

- **现象**：不断提示 `duplicate plugin id detected`。
- **根因**：为了安全起见备份的 `feishu_backup` 文件夹依然留在了 `~/.openclaw/extensions/` 目录下。OpenClaw 的深度扫描机制读取了备份文件，导致同一插件被注册了两次。
- **解决**：将备份文件夹移出 `extensions` 目录，放到用户主目录下，警告消除。

### 5. 飞书官方 SDK 依赖缺失

- **现象**：报错 `Cannot find module '@larksuiteoapi/node-sdk'`。
- **根因**：通过 CLI 下载插件时，未能成功触发 NPM 的依赖安装钩子，导致缺少飞书官方的 Node.js 通信包。
- **解决**：进入全局插件目录 `/usr/lib/node_modules/openclaw/extensions/feishu/`，手动执行 `npm install @larksuiteoapi/node-sdk` 补齐依赖。

### 6. Kimi 逆向接口风控拦截

- **现象**：飞书通道已通，但对话时报错 `API rate limit reached` 和 `访问的 API 暂未开放`。
- **根因**：使用的 `kimi-claw` 插件采用了桥接（Bridge）模式，调用了非公开的内测模型（k2p5）和逆向接口（api.kimi.com/coding）。由于 Token 过期或触发了 Moonshot 官方的风控限制，请求被直接拦截。
- **解决**：修复桥接 Token，或直接切换至稳定合规的 Kimi 官方开放平台 API。

## 第二部分：全新安装的最佳实践 (Golden Path)

基于今天的踩坑经验，如果你需要在新服务器上重新部署一套"OpenClaw + 飞书 + Kimi"的机器人，请完全抛弃之前的修补思路，直接按照以下最简步骤操作。这套流程可以避开今天遇到的所有坑。

### 环境准备

确保服务器已安装 Node.js v22.x 或更高版本。

### Step 1: 安装最新版 OpenClaw 核心

**关键点**：必须确保核心版本 `>=2026.3.1`，否则无法兼容最新的飞书插件。

```bash
npm install -g openclaw@latest
openclaw --version  # 确认 >=2026.3.1
```

### Step 2: 安装飞书插件并补齐依赖

**关键点**：安装后主动进入目录补齐飞书官方 SDK，防止运行时报错。

```bash
openclaw plugins install @openclaw/feishu
# 补齐依赖（防止 npm 钩子未触发）
cd /usr/lib/node_modules/openclaw/extensions/feishu/
npm install @larksuiteoapi/node-sdk
```

### Step 3: 配置飞书通道

直接使用命令行写入配置，避免手动修改 JSON 导致格式错误或死锁。

```bash
openclaw channels add  # 选择 Feishu，输入 App ID 和 App Secret
```

### Step 4: 配置大模型 (强烈建议使用官方 API)

**关键点**：放弃不稳定的网页桥接插件（如 kimi-claw），直接使用 OpenClaw 自带的标准模型通道对接 Kimi 官方 API。

```bash
openclaw models add  # 添加模型配置
```

### Step 5: 诊断与启动

让 OpenClaw 自动检查一遍配置树，修复潜在的系统服务注册问题，然后启动网关。

```bash
openclaw doctor --fix
openclaw gateway start
openclaw status  # 确认一切正常
```

🎉 按照这套最佳实践，整个部署过程只需不到 3 分钟，且不会出现任何版本冲突或依赖丢失的问题。
