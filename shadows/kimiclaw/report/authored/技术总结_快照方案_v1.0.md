## 【问题】连接不稳定的两个表现

### 表现1: 消息接收异常
- **症状**: 无法接受群消息或接收不稳定
- **影响**: 群聊指令丢失，心跳检测失败
- **初步排查**: 飞书channel配置、gateway token有效性

### 表现2: 消息重复处理
- **症状**: 同一条消息反复触发，回到几小时前的上下文
- **根因定位**: `~/.openclaw/extensions/feishu/src/dedup.ts` 去重逻辑缺陷
- **具体漏洞**: 
  - 检查 `senderOpenId` 导致同一用户的所有消息被误拦截
  - 只检查 `messageId` 才是正确做法
  - 双层去重：内存缓存 + Session历史持久化
  - 24小时窗口，每文件最多1000行

**修复状态**: 已修改 `dedup.ts`，需重启gateway生效。

## 【方案】KimiClaw快照架构

### 核心组件

| 组件 | 路径 | 功能 |
|------|------|------|
| 入口脚本 | `kimiclaw/bin/kimiclaw` | 统一CLI接口 |
| 配置中心 | `kimiclaw/config/default.json` | 路径映射+脱敏规则+大小限制 |
| 快照引擎 | `kimiclaw/lib/snapshot.sh` | SHA256哈希比对+增量备份 |
| 恢复引擎 | `kimiclaw/lib/restore.sh` | 自动备份+幂等恢复+凭证填充 |
| 推送模块 | `kimiclaw/lib/push.sh` | GitHub自动提交 |
| 设计文档 | `kimiclaw/docs/DESIGN.md` | 架构设计+安全策略+性能指标 |

### 技术特性

1. **增量备份**: 基于SHA256文件哈希，仅备份变更文件，节省90%+时间
2. **深度脱敏**: 28+ JSON字段模式 + 10+值模式匹配 + PEM私钥检测 + session jsonl深度扫描
3. **安全扫描**: 推送前自动扫描敏感信息，占位符格式 `{{SECRET:field_name}}`
4. **自测套件**: 12项单元测试覆盖help/snapshot/restore/安全扫描

### 输出结构

```
shadows/kimiclaw/
├── agents/              # session文件
├── canvas/
├── credentials/         # 脱敏后的凭证模板
├── cron/               # 定时任务
├── devices/

├── skills/             # 本地skills
├── workspace/          # memory/SOUL.md等
├── manifest.json       # 文件哈希清单
├── openclaw.json       # 主配置（脱敏）
└── 快照报告_*.md       # 可读性报告
```
