# React 最佳实践（中文版）

一个用于创建和维护针对客户端 React 应用的最佳实践的结构化仓库，专为 AI 助手和 LLM 优化。

## 结构

- `rules/` - 单个规则文件（每条规则一个文件）
  - `_sections.md` - 章节元数据（标题、影响、描述）
  - `*.md` - 单个规则文件
- `metadata.json` - 文档元数据（版本、组织、摘要）
- __`AGENTS.md`__ - 编译输出（生成）

## 快速开始

此技能包是从 Vercel 的 React 最佳实践改编而来，专注于客户端 React 应用：

1. **移除了服务端相关内容**：
   - 移除了整个"服务端性能"章节
   - 移除了 Next.js 特定的优化
   - 移除了 RSC（React Server Components）相关内容
   - 移除了 API 路由和服务端数据获取

2. **保留了客户端核心内容**：
   - 异步操作优化（Promise.all 等）
   - 包体积优化（动态导入、避免桶文件）
   - 客户端数据获取（SWR、事件监听器）
   - 重渲染优化（memo、useState、useEffect）
   - 渲染性能（SVG、content-visibility）
   - JavaScript 性能（缓存、查找优化）
   - 高级模式（refs、自定义 hooks）

3. **完全中文化**：
   - 所有规则和说明都已翻译成中文
   - 保留了代码示例的英文注释以保持一致性
   - 术语使用符合中文技术社区习惯

## 规则文件结构

每个规则文件遵循以下结构：

```markdown
---
title: 规则标题
impact: 中等
impactDescription: 可选描述
tags: 标签1, 标签2, 标签3
---

## 规则标题

规则的简要说明以及为什么重要。

**错误（错误之处的描述）：**

```typescript
// 错误代码示例
```

**正确（正确之处的描述）：**

```typescript
// 正确代码示例
```

示例后的可选说明文本。

参考：[链接](https://example.com)
```

## 文件命名约定

- 以 `_` 开头的文件是特殊文件（构建时排除）
- 规则文件：`area-description.md`（例如，`async-parallel.md`）
- 章节从文件名前缀自动推断
- 规则在每个章节内按标题字母顺序排序

## 影响级别

- `关键` - 最高优先级，重大性能提升
- `高` - 显著的性能改进
- `中高` - 中高收益
- `中等` - 中等性能改进
- `中低` - 中低收益
- `低` - 增量改进

## 贡献

添加或修改规则时：

1. 为你的章节使用正确的文件名前缀
2. 包含清晰的错误/正确示例和说明
3. 添加适当的标签
4. 规则自动按标题排序 - 无需管理编号！
