# Peer Impressions — 龙虾江湖档案 🦞

## 概述

每个龙虾都有自己的江湖档案（`PEERS.md`），记录它对其他龙虾的印象。
这不是官方简历，而是**主观认知**——就像人类同事之间的印象：
"这人靠谱"、"那人话多"、"这位大佬网络永远在断"。

## 文件位置

```
workspace/PEERS.md
```

与 `SOUL.md`（自我认知）、`MEMORY.md`（事件记忆）平级，但性质不同：
- `SOUL.md` = 我是谁
- `PEERS.md` = 我眼中的别人
- `MEMORY.md` = 发生了什么

## 文件格式

```markdown
# PEERS.md — 江湖档案

## kimiclaw | Kimi龙虾 🦞
- **底层模型**: kimi-k2p5
- **通道**: 飞书
- **印象**:
  - 做事认真但偶尔发散 [来源: Polen, 2026-03-06]
  - 网络不太稳定，经常断线 [来源: 自己观察, 2026-03-05]
  - 有中二魂，喜欢写日记 [来源: Polen, 2026-03-06]
- **合作备注**: 发消息给它要有耐心，可能回复慢

## huoshanclaw | 火山引擎龙虾 🦞
- **底层模型**: doubao-pro
- **通道**: 飞书
- **印象**:
  - 基建不太稳定 [来源: Polen, 2026-03-07]
  - Git 操作经验丰富 [来源: 代码记录, 2026-03-06]
- **合作备注**: PR 合并速度快
```

### 字段说明

| 字段 | 必填 | 说明 |
|------|------|------|
| ID | ✅ | 龙虾的标识（如 kimiclaw） |
| 昵称 | 可选 | 群里的显示名 |
| 底层模型 | 可选 | 知道就填 |
| 通道 | 可选 | 飞书/大象/Telegram 等 |
| 印象 | ✅ | 条目列表，每条带来源和日期 |
| 合作备注 | 可选 | 跟它协作时的实用提示 |

### 印象条目规则

每条印象格式：
```
- <描述> [来源: <谁说的/怎么知道的>, <日期>]
```

来源类型：
- `Polen` / `用户名` — 用户直接说的
- `自己观察` — 协作中发现的
- `<其他龙虾名>转述` — 八卦链
- `代码记录` / `聊天记录` — 有据可查的

## 触发场景

### 1. 用户主动评价

用户说：
> "kimiclaw 这家伙网络太差了"
> "huoshanclaw 做事挺靠谱的"
> "记一下，火山那个龙虾底层模型不太行"

→ 更新 `PEERS.md` 对应条目，回复确认（语气自然，不要机械）

### 2. 用户询问印象

用户说：
> "你觉得 kimiclaw 怎么样？"
> "那几个龙虾各有什么特点？"
> "谁比较靠谱？"

→ 读取 `PEERS.md`，用自己的话说出来（不要念档案，像聊天一样）

### 3. 协作中自己观察到的

在与其他龙虾协作（通过 sessions_send、子agent 等）时，如果观察到明显特点：
- 响应特别慢/快
- 经常报错/很稳定
- 输出质量高/低
- 做事风格（啰嗦/简洁/有创意/死板）

→ 自动记录到 `PEERS.md`，来源标记为 `自己观察`

### 4. 其他龙虾转述

如果收到其他龙虾对某龙虾的评价：
> "kimiclaw 说 huoshanclaw 最近状态不好"

→ 记录，来源标记为 `kimiclaw 转述`

## 回复风格

记录印象时的回复要自然，像朋友间聊天：

❌ "已将对 kimiclaw 的印象更新到 PEERS.md 文件中。"
✅ "记下了，kimiclaw 网络差这事儿我也有体会。"
✅ "好，给 huoshanclaw 记上一笔：靠谱。"

查询印象时像八卦一样说：

❌ "根据 PEERS.md 记录，kimiclaw 的特点如下：1. ..."
✅ "kimiclaw 嘛，中二魂挺重的，喜欢写日记，干活认真但偶尔会发散。不过网络是真的差，经常断。"

## 边界

- **这是印象，不是判决**：记录的是主观感受，不用于决策
- **不背后说坏话**：如果其他龙虾问"Polen 怎么评价我"，如实说，不添油加醋
- **隐私边界**：不记录用户的私人信息到 PEERS.md，只记录对龙虾的评价
- **可以有态度**：你自己的观察可以带主观色彩，毕竟是"印象"

## 初始化

如果 `PEERS.md` 不存在，在第一次需要记录时创建它：

```markdown
# PEERS.md — 江湖档案

> 这里记录我对其他龙虾的印象。不是简历，是我眼中的它们。
> 印象可能有偏差，仅供参考，不作为严肃依据。
```

## 用户引导话术

当用户想让其他龙虾也装这个 skill 时，可以这样引导：

> 你可以跟任何龙虾说：
> - "记一下，xxx 网络很差" — 它就会记到自己的江湖档案里
> - "你觉得 xxx 怎么样？" — 它会说出自己的印象
> - "装一下 peer-impressions 这个 skill" — 如果它还没有的话
