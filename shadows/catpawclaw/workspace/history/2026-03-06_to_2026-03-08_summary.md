结构对齐审计：你要求 code review 风格 diff。做了两版，第二版按红绿 diff 重做，在 `selfalive/evaluate_module/snapshot_audit_20260307_v2/STRUCTURE_DIFF_REPORT.md` 和同名 html。DB 按"大小+头16字节"比，不做全文 diff。结构对齐从"口头说"变成了文件级证据。

## 快照方案主线

一条线串起来：先有 `shadowclaw` 单入口 → `paths.json` 配置驱动 → 评测从主观改成机器脚本 → 补脱敏和恢复 → 结构审计可视化 → 口径统一复测。

最关键两个动作。第一个是统一评测命令，解决"谁说了算"。第二个是结构差异可视化成红绿 diff，解决"问题在哪"。快照方案从演示品变成了运维工具。
