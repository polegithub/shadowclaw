### 分析

三个方案对比：
- **KimiClaw**: CLI 工具好（bin/kimiclaw 入口），脱敏机制成熟，但目录覆盖不全（缺 identity/, .env, workspace/memory/），文件嵌套太深（snapshot 里套 snapshot）
- **HuoshanClaw**: 目录覆盖最完整（identity/, .env, memory/main.sqlite, feishu-pairing 等），有真实示例数据，但纯文档无脚本
- **CatClaw v1.0**: 试图融合两者，但直接复制了 kimiclaw 的 skills（cron-executor, message-dedup），导致 30+ 文件臃肿

### 重构内容

CatClaw v2.0 核心改变：
1. **单脚本入口** `bin/shadowclaw` — snapshot/restore/push/verify 四合一
2. **配置驱动** `config/paths.json` — 路径、优先级、大小限制、脱敏规则全在配置中
3. **扩大覆盖** — 新增 `~/.openclaw/skills/`, `feishu/`, `devices/` 备份
4. **移除冗余** — 删除所有复制来的运行时 skills 和重复记忆文件
5. **新增 verify** — 检查快照完整性和安全性
6. 从 30+ 文件 → 4 个文件
